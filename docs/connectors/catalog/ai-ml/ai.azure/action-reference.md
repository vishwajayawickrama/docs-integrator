# Actions

The `ballerinax/ai.azure` package exposes the following clients:

| Client | Purpose |
|--------|---------|
| [`OpenAI Model Provider`](#openai-model-provider) | Provides chat completion and structured generation using Azure-hosted OpenAI language models. |
| [`Embedding Provider`](#embedding-provider) | Generates text embeddings using Azure-hosted OpenAI embedding models. |
| [`AI Search Knowledge Base`](#ai-search-knowledge-base) | Implements the ai:KnowledgeBase interface for Azure AI Search — document ingestion, vector retrieval, and deletion. |

---

## OpenAI model provider

Provides chat completion and structured generation using Azure-hosted OpenAI language models.

### Configuration

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `serviceUrl` | `string` | Required | The base URL of the Azure OpenAI API endpoint (e.g., `https://your-resource.openai.azure.com/`). |
| `apiKey` | `string` | Required | The Azure OpenAI API key for authentication. |
| `deploymentId` | `string` | Required | The deployment identifier for the specific model deployment in Azure. |
| `apiVersion` | `string` | Required | The Azure OpenAI API version (e.g., `"2024-06-01"`). |
| `maxTokens` | `int` | `512` | The upper limit for the number of tokens in the response generated by the model. |
| `temperature` | `decimal` | `0.7` | Controls randomness in the model's output. Lower values produce more deterministic results. |
| `connectionConfig` | `ConnectionConfig` | `{}` | Additional HTTP connection configuration (HTTP version, timeout, retry, SSL, proxy, etc.). |

### Initializing the client

```ballerina
import ballerina/ai;
import ballerinax/ai.azure;

configurable string serviceUrl = ?;
configurable string apiKey = ?;
configurable string deploymentId = ?;
configurable string apiVersion = ?;

final ai:ModelProvider azureModel = check new azure:OpenAiModelProvider(
    serviceUrl,
    apiKey,
    deploymentId,
    apiVersion
);
```

### Operations

#### Chat completion

<details>
<summary>chat</summary>

Sends a list of chat messages to the Azure OpenAI model and returns an assistant response. Supports tool/function calling when tool definitions are provided.

Parameters:

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `messages` | `ai:ChatMessage[]\|ai:ChatUserMessage` | Yes | A list of chat messages (user, assistant, system, function) or a single user message. |
| `tools` | `ai:ChatCompletionFunctions[]` | Yes | Tool definitions available for the model to invoke. Pass `[]` if no tools are needed. |
| `stop` | `string?` | No | A stop sequence that causes the model to stop generating. Defaults to `()`. |

Returns: `ai:ChatAssistantMessage|ai:Error`

Sample code:

```ballerina
ai:ChatMessage[] chatMessages = [
    {role: "user", content: "What is the capital of France?"}
];
ai:ChatAssistantMessage response = check azureModel->chat(chatMessages, tools = []);
```

Sample response:

```ballerina
{"role": "assistant", "content": "The capital of France is Paris.", "toolCalls": null}
```

</details>

#### Structured generation

<details>
<summary>generate</summary>

Sends a natural language prompt to the model and returns a typed Ballerina value matching the specified type descriptor. Uses tool-based extraction internally to produce structured output.

Parameters:

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `prompt` | `ai:Prompt` | Yes | A tagged template literal prompt that can include text and interpolated values. |
| `td` | `typedesc<anydata>` | No | Type descriptor specifying the expected return type (e.g., `int`, `string`, a record type). Inferred from the call site. |

Returns: `td|ai:Error`

Sample code:

```ballerina
type Review record {|
    string title;
    int rating;
    string summary;
|};

Review result = check azureModel->generate(`Rate this blog out of 10.
    Title: "Ballerina for Integration"
    Content: "Ballerina is a cloud-native programming language..."`);
```

Sample response:

```ballerina
{"title": "Ballerina for Integration", "rating": 8, "summary": "A well-written introduction to Ballerina's integration capabilities."}
```

</details>

## Embedding provider

Generates text embeddings using Azure-hosted OpenAI embedding models.

### Configuration

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `serviceUrl` | `string` | Required | The base URL of the Azure OpenAI API endpoint. |
| `accessToken` | `string` | Required | The access token (API key) for authenticating API requests. |
| `apiVersion` | `string` | Required | The API version of the Azure OpenAI API. |
| `deploymentId` | `string` | Required | The deployment ID of the embedding model. |
| `config` | `ConnectionConfig` | `{}` | Additional HTTP connection configuration (HTTP version, timeout, retry, SSL, proxy, etc.). |

### Initializing the client

```ballerina
import ballerina/ai;
import ballerinax/ai.azure;

configurable string serviceUrl = ?;
configurable string accessToken = ?;
configurable string deploymentId = ?;
configurable string apiVersion = ?;

final ai:EmbeddingProvider embeddingProvider = check new azure:EmbeddingProvider(
    serviceUrl,
    accessToken,
    apiVersion,
    deploymentId
);
```

### Operations

#### Embedding operations

<details>
<summary>embed</summary>

Generates an embedding vector for a single text chunk.

Parameters:

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `chunk` | `ai:Chunk` | Yes | The chunk containing the text content to embed. Must be a `TextDocument` or `TextChunk`. |

Returns: `ai:Embedding|ai:Error`

Sample code:

```ballerina
ai:TextChunk chunk = {
    content: "Ballerina is an open-source programming language for cloud-native integration."
};
ai:Embedding embedding = check embeddingProvider->embed(chunk);
```

Sample response:

```ballerina
{"id": "vec-001", "embedding": [0.0023, -0.0091, 0.0152, -0.0034, 0.0087], "chunk": {"content": "Ballerina is an open-source programming language for cloud-native integration."}}
```

</details>

<details>
<summary>batchEmbed</summary>

Generates embedding vectors for multiple text chunks in a single batch request.

Parameters:

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `chunks` | `ai:Chunk[]` | Yes | An array of chunks to be converted into embeddings. |

Returns: `ai:Embedding[]|ai:Error`

Sample code:

```ballerina
ai:TextChunk[] chunks = [
    {content: "Ballerina is a cloud-native programming language."},
    {content: "Azure OpenAI provides enterprise-grade AI models."}
];
ai:Embedding[] embeddings = check embeddingProvider->batchEmbed(chunks);
```

Sample response:

```ballerina
[
  {"id": "vec-001", "embedding": [0.0023, -0.0091, 0.0152], "chunk": {"content": "Ballerina is a cloud-native programming language."}},
  {"id": "vec-002", "embedding": [0.0145, -0.0032, 0.0078], "chunk": {"content": "Azure OpenAI provides enterprise-grade AI models."}}
]
```

</details>

## AI search knowledge base

Implements the ai:KnowledgeBase interface for Azure AI Search — document ingestion, vector retrieval, and deletion.

### Configuration

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `serviceUrl` | `string` | Required | The Azure AI Search service endpoint URL (e.g., `https://your-search.search.windows.net`). |
| `apiKey` | `string` | Required | The Azure AI Search admin API key for authentication. |
| `index` | `string\|search:SearchIndex` | Required | The name of an existing search index or a `SearchIndex` definition to create a new index. |
| `embeddingModel` | `ai:EmbeddingProvider?` | `()` | An optional embedding provider for generating vector embeddings during ingestion and retrieval. |
| `chunker` | `ai:Chunker\|ai:AUTO\|ai:DISABLE` | `ai:AUTO` | Document chunking strategy. `ai:AUTO` auto-detects format, `ai:DISABLE` skips chunking, or provide a custom `ai:Chunker`. |
| `verbose` | `boolean` | `false` | Enable verbose logging for debugging. |
| `apiVersion` | `string` | `"2025-09-01"` | The Azure AI Search API version. |
| `contentFieldName` | `string` | `"content"` | The name of the main content field in the search index. |
| `searchClientConnectionConfig` | `search:ConnectionConfig` | `{}` | HTTP connection configuration for the search service client. |
| `indexClientConnectionConfig` | `index:ConnectionConfig` | `{}` | HTTP connection configuration for the index client. |
| `semanticConfigurationName` | `string?` | `()` | The name of the semantic search configuration to use for hybrid or semantic queries. |

### Initializing the client

```ballerina
import ballerina/ai;
import ballerinax/ai.azure;

configurable string searchServiceUrl = ?;
configurable string searchApiKey = ?;
configurable string indexName = ?;

// Optionally provide an embedding model for vector search
configurable string openaiServiceUrl = ?;
configurable string openaiApiKey = ?;
configurable string embeddingDeploymentId = ?;
configurable string apiVersion = ?;

final ai:EmbeddingProvider embeddingModel = check new azure:EmbeddingProvider(
    openaiServiceUrl,
    openaiApiKey,
    apiVersion,
    embeddingDeploymentId
);

final ai:KnowledgeBase knowledgeBase = check new azure:AiSearchKnowledgeBase(
    searchServiceUrl,
    searchApiKey,
    indexName,
    embeddingModel = embeddingModel
);
```

### Operations

#### Document operations

<details>
<summary>ingest</summary>

Ingests documents or chunks into the Azure AI Search index. Automatically chunks documents (if enabled), generates embeddings (if an embedding model is provided), and uploads to the index.

Parameters:

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `documents` | `ai:Chunk[]\|ai:Document[]\|ai:Document` | Yes | One or more documents or pre-chunked content to ingest into the knowledge base. |

Returns: `ai:Error?`

Sample code:

```ballerina
ai:TextDocument[] documents = [
    {
        content: "Ballerina is an open-source cloud-native programming language.",
        metadata: {"source": "docs", "topic": "overview"}
    },
    {
        content: "Azure AI Search provides enterprise-grade search capabilities.",
        metadata: {"source": "docs", "topic": "search"}
    }
];
check knowledgeBase->ingest(documents);
```

</details>

<details>
<summary>retrieve</summary>

Queries the Azure AI Search index and returns the top matching results based on vector similarity and/or semantic search.

Parameters:

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `query` | `string` | Yes | The search query string. |
| `maxLimit` | `int` | No | The maximum number of results to return. Defaults to `10`. |
| `filters` | `ai:MetadataFilters?` | No | Optional metadata filters to narrow the search results. |

Returns: `ai:QueryMatch[]|ai:Error`

Sample code:

```ballerina
ai:QueryMatch[] results = check knowledgeBase->retrieve("cloud-native programming", maxLimit = 5);
```

Sample response:

```ballerina
[
  {
    "id": "doc-001",
    "chunk": {
      "content": "Ballerina is an open-source cloud-native programming language.",
      "metadata": {"source": "docs", "topic": "overview"}
    },
    "similarityScore": 0.95
  },
  {
    "id": "doc-002",
    "chunk": {
      "content": "Azure AI Search provides enterprise-grade search capabilities.",
      "metadata": {"source": "docs", "topic": "search"}
    },
    "similarityScore": 0.72
  }
]
```

</details>

<details>
<summary>deleteByFilter</summary>

Deletes documents from the Azure AI Search index that match the specified metadata filters.

Parameters:

| Name | Type | Required | Description |
|------|------|----------|-------------|
| `filters` | `ai:MetadataFilters` | Yes | Metadata filters identifying the documents to delete. |

Returns: `ai:Error?`

Sample code:

```ballerina
check knowledgeBase->deleteByFilter({
    filters: [
        {key: "source", value: "docs", operator: ai:EQ}
    ],
    logicalOperator: ai:AND
});
```

</details>
