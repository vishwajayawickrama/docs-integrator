# Why WSO2 Integrator

Connect AI agents, APIs, data, and events across cloud, on-premises, and hybrid environments. Build any type of integration and AI agent with the 100% open-source WSO2 Integrator.

## Build any type of AI agent or integration

Orchestrate intelligent flows by combining built-in capabilities with a wide array of connectors to seamlessly integrate data, APIs, and Model Context Protocol (MCP) servers.

- Automation: Streamline repetitive workflows and improve operational efficiency.
- AI agents and GenAI apps: Build AI agents that use tools, maintain memory, and orchestrate multi-step workflows. Connect AI agents to your preferred models, knowledge bases, and tools.
- Integration as an API: Expose your integrations securely to internal and external consumers.
- Event-driven integration: React to events in real time across your distributed architecture.
- File-driven integration: Process batches and large files efficiently and reliably.

## Key differentiators

Build integrations, AI agents, and GenAI apps faster, easier, and better with these key capabilities:

### AI-assisted development
Boost productivity and accelerate time-to-market with an AI-assisted development experience. It includes interactive development with code generation, suggestions, and modifications using natural language, AI-assisted testing, and more. AI isn't a bolt-on feature. It's woven into every stage, including AI-assisted data mapping and natural language functions that call large language models (LLMs) as typed functions.

### Visual designer and code editor
Unlike platforms that force a choice between a visual tool or a coding experience, WSO2 Integrator provides both in the same environment, always in sync. Build flows visually, drop into code when you need precision, and switch back at any point without losing work.

### Visual data mapper and format support
Visually map and transform complex data structures between formats — JSON, XML, EDI, CSV, and more — without writing transformation code by hand. The built-in data mapper shows source and target schemas side by side and generates the mapping logic for you, making it straightforward to bridge systems that speak different data languages.

### Ballerina-powered
Built on Ballerina, a language designed specifically for integration. You get compile-time type safety for data transformations, built-in network primitives, and sequence diagram visualization—capabilities that general-purpose languages can't match.

### More than 600 connectors
Pre-built connectors for Software as a Service (SaaS) applications, databases, messaging systems, cloud services, AI and LLM providers, and protocols. You also gain the ability to use any Ballerina package from Ballerina Central.

### Migration tooling
Built-in tools help you migrate integrations from legacy platforms such as WSO2 Micro Integrator and other ESBs. Move existing flows to WSO2 Integrator incrementally, at your own pace, without having to rewrite everything from scratch.

### Built-in observability
Monitor integrations and agents continuously with built-in metrics, logs, and traces. Get centralized visibility across your entire ecosystem so you can detect and diagnose issues before they affect users, without setting up a separate observability stack.

## Run it your way

WSO2 Integrator is 100% open-source and the only integration platform available as both self-hostable software and a fully managed cloud service. If you need complete data sovereignty, deploy to your own servers, bare metal, or private cloud — the same integration code targets Docker, Kubernetes, AWS, Azure, GCP, and air-gapped environments without modification. Bring your own CI/CD pipeline and maintain full control of your stack from day one.

If you prefer to manage zero infrastructure, WSO2's Devant iPaaS has you covered. WSO2 handles provisioning, upgrades, scaling, and availability so your team can focus on building integrations rather than operating them. You get flexible data plane deployment, centralized observability, continuous updates with zero downtime, and multi-region availability out of the box.

## Who it's for

- Integration developers: building APIs, event-driven flows, file processing, and automations.
- Platform engineers: standardizing integration infrastructure across teams.
- Teams modernizing from legacy ESBs: looking for a clean break from XML-heavy, proprietary tooling.
