## New relic module

Observability extension that automatically registers a New Relic metrics reporter and an OpenTelemetry trace provider with the Ballerina runtime on module initialization.


### Configuration (`ModuleConfiguration (Config.toml)`)

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `apiKey` | `string\|string[]` | Required | New Relic License Key (Ingest key). Accepts a single string or an array of strings to fan out to multiple accounts. Can also be supplied via the `BALLERINA_NEW_RELIC_API_KEY` environment variable (comma-separated for multiple keys); the environment variable takes precedence over `Config.toml`. |
| `region` | `string` | `"us"` | Target New Relic data center region. Accepted values: `"us"` or `"eu"`. |
| `isTraceLoggingEnabled` | `boolean` | `false` | Enable trace-level logging for the New Relic reporter internals. |
| `isPayloadLoggingEnabled` | `boolean` | `false` | Log the raw telemetry payloads sent to New Relic (useful for debugging). |
| `tracingSamplerType` | `string` | `"const"` | Trace sampling strategy. One of `"const"` (sample all or none based on param), `"ratelimiting"` (limit traces per second), or `"probabilistic"` (sample by probability). |
| `tracingSamplerParam` | `decimal` | `1` | Parameter for the sampler. For `"const"`: `1` = sample all, `0` = sample none. For `"ratelimiting"`: max traces per second. For `"probabilistic"`: probability (0.0–1.0). |
| `tracingReporterFlushInterval` | `int` | `15000` | Interval in milliseconds at which buffered spans are flushed to New Relic. |
| `tracingReporterBufferSize` | `int` | `10000` | Maximum number of spans held in the in-memory buffer before flushing. |
| `metricReporterFlushInterval` | `int` | `15000` | Interval in milliseconds at which collected metrics are flushed to New Relic. |
| `metricReporterClientTimeout` | `int` | `10000` | HTTP client timeout in milliseconds for metric POST requests to New Relic. |
| `additionalAttributes` | `map<string>` | `{}` | Static key-value pairs added as custom attributes to every exported metric. |

### Initializing the client

```ballerina
// Add the dependency in Ballerina.toml:
// [[dependency]]
// org = "ballerinax"
// name = "newrelic"
// version = "1.0.3"

// Side-effect import — no client variable needed
import ballerinax/newrelic as _;

// Configure in Config.toml:
// [ballerinax.newrelic]
// apiKey = "<NEW_RELIC_LICENSE_KEY>"
// region = "us"
```

### Operations

#### Metrics reporting

<details>
<summary>HTTP service metrics export</summary>

<div>

Automatically collects per-service, per-resource HTTP metrics from the Ballerina runtime and pushes them to the New Relic Metric API on every flush interval. Metrics include response time statistics, request counts, error counts, and in-flight request gauges — all tagged with service name, resource path, and any `additionalAttributes`.



Returns: `error?`

Sample code:

```ballerina
import ballerina/http;
import ballerinax/newrelic as _;

// Metrics for this service are automatically exported to New Relic.
// No additional code is required.
service /order on new http:Listener(9090) {
    resource function get status() returns string {
        return "OK";
    }
}
```

</div>

</details>

<details>
<summary>Kafka metrics export</summary>

<div>

Automatically tracks Kafka publisher counts, consumer counts, and publish error counts exposed by the Ballerina Kafka module, and includes them in the periodic metric flush to New Relic.



Returns: `error?`

Sample code:

```ballerina
import ballerinax/newrelic as _;
// Kafka metrics are collected automatically when using ballerinax/kafka.
// Exported metrics include:
//   kafka_publishers_value
//   kafka_consumers_value
//   kafka_errors_value
```

</div>

</details>

#### Distributed tracing

<details>
<summary>OpenTelemetry trace export</summary>

<div>

Registers an OpenTelemetry-compatible trace provider that captures distributed spans generated by the Ballerina runtime and exports them to New Relic's OTLP endpoint (`otlp.nr-data.net:4317` for US, `otlp.eu01.nr-data.net:4317` for EU) using the configured sampling strategy and flush settings.



Returns: `error?`

Sample code:

```ballerina
import ballerina/http;
import ballerinax/newrelic as _;

// Distributed traces for inter-service calls are captured and exported automatically.
http:Client inventoryClient = check new ("http://inventory-svc:8080");

service /checkout on new http:Listener(9091) {
    resource function post place(http:Caller caller, http:Request req) returns error? {
        // This outbound call is automatically traced as a child span.
        http:Response res = check inventoryClient->get("/items");
        check caller->respond(res);
    }
}
```

</div>

</details>

#### Multi-account telemetry

<details>
<summary>Fan-out to multiple New Relic accounts</summary>

<div>

When `apiKey` is set to an array of License Keys (or multiple keys are supplied via `BALLERINA_NEW_RELIC_API_KEY`), the module fans out every metric and trace payload to all configured New Relic accounts simultaneously.



Returns: `error?`

Sample code:

```ballerina
// Config.toml — fan out to two accounts:
// [ballerinax.newrelic]
// apiKey = ["<LICENSE_KEY_PROD>", "<LICENSE_KEY_STAGING>"]
// region = "us"

import ballerinax/newrelic as _;
// All telemetry is now mirrored to both accounts automatically.
```

</div>

</details>
